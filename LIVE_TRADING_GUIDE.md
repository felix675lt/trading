# 실거래 준비 가이드 (Live Trading Checklist)

## 🚀 페이퍼 → 실거래 전환 완전 가이드

---

## ✅ Phase 1: 기술적 준비 (1-2시간)

### 1.1 API 키 확보
```bash
# Binance 선물 거래 (권장)
필요한 권한:
  ✓ Futures Trading (읽기/쓰기)
  ✓ Read Access to Account Data
  ✗ Withdraw (비활성화 - 안전)
  ✗ IP Whitelist 설정 권장

테스트넷 설정 (선택사항):
  # testnet.binancefuture.com 먼저 테스트
```

### 1.2 환경변수 설정
```bash
# ~/.bashrc 또는 ~/.zshrc에 추가
export BINANCE_API_KEY="your_actual_key"
export BINANCE_SECRET="your_actual_secret"

# 확인
echo $BINANCE_API_KEY  # 출력되면 OK
```

### 1.3 설정 파일 변경
```yaml
# config/default.yaml

trading:
  symbols:
    - "BTC/USDT:USDT"    # 시작은 BTC 단일만
    - "ETH/USDT:USDT"    # 2주 후에 추가
  mode: "live"            # ← "paper"에서 변경
  leverage: 3             # ← 5에서 3으로 낮춤 (안전)

risk:
  max_position_pct: 0.05  # ← 10%에서 5%로 낮춤
  max_daily_loss_pct: 0.02  # ← 5%에서 2%로 낮춤
  max_drawdown_pct: 0.10  # ← 15%에서 10%로 낮춤
```

### 1.4 시스템 검증
```bash
# 설정 유효성 확인
python3 -c "import yaml; yaml.safe_load(open('config/default.yaml'))"

# API 키 로드 확인
python3 -c "import os; print('API KEY:', len(os.environ.get('BINANCE_API_KEY','')))"

# 로그 디렉토리 생성
mkdir -p logs
```

## ✅ Phase 2: 페이퍼 모드 최종 테스트 (3-5일)

### 2.1 신호 정상성 검증
```
체크리스트:
  ☐ 대시보드 http://localhost:8888 접속 확인
  ☐ Status: "PAPER MODE" 표시 확인
  ☐ 실시간 로그 업데이트 (30초마다)
  ☐ 외부 신호 표시 (트위터/센티먼트 등)
  ☐ 최소 10회 이상의 "analysis" 로그 확인
```

### 2.2 거래 신호 확인
```
확인 사항:
  ☐ 거래 신호가 생성되는가?
    → "trade_open" 로그 확인
  ☐ 신호의 근거가 합리적인가?
    → "reason" 필드: ML/RL/외부요인 균형 있나?
  ☐ 청산 신호가 생성되는가?
    → "trade_close" 로그 (2시간 이상 보유 후)
  ☐ 수익/손실이 합리적인가?
    → PnL 범위: -2% ~ +4% (페이퍼에서)
```

### 2.3 외부 신호 반응 테스트
```
시뮬레이션:
  1. 크립토 뉴스 검색 (Bitcoin 관련)
  2. 5분 후 대시보드 확인
  3. "External Factors" 신호 변화 관찰
  4. "Live Learning Log"에 "analysis" 로그 추가 확인

✓ 반응 시간: 1-2분 (정상)
✓ 신호 변화: ±0.1~0.3 (정상)
```

### 2.4 자기학습 검증
```
확인 사항:
  ☐ 매 24시간마다 "retrain" 로그?
  ☐ 로그: "자기학습 재훈련 시작" ~ "완료"?
  ☐ BTC/ETH 모두 학습되었나?
  ☐ 모델이 저장되었나? (logs/models/ 확인)
```

## ✅ Phase 3: 리스크 관리 설정 (1시간)

### 3.1 초기 자본 결정
```
권장:
  최소: $1,000 (학습 목적용)
  보통: $5,000 (실험적 거래)
  적정: $10,000+ (장기 운영)

계산:
  평균 거래 크기 = 초기 자본 × 5% × 레버리지
  예: $10,000 × 5% × 3x = $1,500 (양호)
```

### 3.2 손실 제한 설정
```yaml
risk:
  max_daily_loss_pct: 0.02      # 일일 손실 한계: 2% ($200)
  max_drawdown_pct: 0.10         # 최대 낙폭: 10% ($1,000)
  stop_loss_pct: 0.02            # 각 거래 SL: 2%
  take_profit_pct: 0.04          # 각 거래 TP: 4%
```

### 3.3 거래 제한 설정
```
첫 주: (2026-03-15 ~ 03-21)
  ☐ BTC만 거래 (ETH 금지)
  ☐ 최대 1개 동시 포지션
  ☐ 최대 3회/일 거래
  ☐ 시간 제한 없음 (24시간 운영)

첫 달: (03-22 ~ 04-14)
  ☐ BTC + ETH 추가
  ☐ 최대 2개 동시 포지션
  ☐ 최대 5회/일 거래

두 번째 달: (04-15 ~ 05-14)
  ☐ 제한 해제 가능 (성과에 따라)
```

## ✅ Phase 4: 실거래 시작 (수동 확인 1주)

### 4.1 실거래 전환 1시간 전
```bash
# 1단계: 백업
cp -r . backup_$(date +%Y%m%d_%H%M%S)/

# 2단계: 로그 확인
tail -f logs/autotrader.log

# 3단계: 환경변수 확인
echo $BINANCE_API_KEY | wc -c  # 50자 이상이어야 함

# 4단계: API 연결 테스트 (선택)
python3 -c "from binance import AsyncClient; print('Binance 라이브러리 OK')"
```

### 4.2 실거래 시작
```bash
# 5단계: 시스템 시작
python3 main.py config/default.yaml

# 6단계: 대시보드 모니터링
# 터미널 2: curl http://localhost:8888/api/status
# 상태: "LIVE MODE" 확인

# 7단계: 거래 모니터링
# 터미널 3: tail -f logs/autotrader.log | grep -E "(trade_open|trade_close)"
```

### 4.3 수동 검증 프로토콜 (첫 1주일)

**매 거래마다 확인:**
```
trade_open 로그 발생:
  ☐ 신호가 합리적인가? (reason 확인)
  ☐ 가격이 최근 시장가 맞나?
  ☐ 크기가 위험 관리 내인가?
  ☐ 레버리지가 설정값 맞나?

trade_close 로그:
  ☐ 청산이 자동인가? (SL/TP 도달)
  ☐ PnL이 합리적인가?
  ☐ 거래 기간이 설정값 맞나?
```

**매일 아침 검증:**
```
대시보드:
  ☐ Equity 변화 확인 (±5% 내)
  ☐ Daily PnL 확인
  ☐ Risk Status 확인 (ACTIVE)
  ☐ 거래 개수 확인 (1-5회/일)

로그:
  ☐ ERROR 없음?
  ☐ 외부 신호 갱신?
  ☐ RL 모델 예측 정상?
```

## ✅ Phase 5: 점진적 확대 (2주~2개월)

### 5.1 성과 모니터링 기준
```
1주 후:
  최소 목표: 손실 없음 (Break-even)
  우수 목표: +2% 이상 (Sharpe ratio > 0.5)
  ✓ 달성: 2주 더 진행
  ✗ 미달: 신호 분석 + 1주 연장

2주 후:
  최소 목표: +1% 이상
  우수 목표: +5% 이상
  ✓ 달성: 제한 해제 시작
  ✗ 미달: 전략 재검토
```

### 5.2 확대 계획
```
Week 1-2:
  ☐ BTC 단일 (최대 1 포지션)
  ☐ 레버리지 2x (보수)

Week 3-4:
  ☐ BTC + ETH (최대 2 포지션)
  ☐ 레버리지 3x (목표)

Month 2:
  ☐ BTC + ETH (최대 2 포지션)
  ☐ 레버리지 5x (공격) - 성과 기반

Month 3+:
  ☐ 자유 거래 (risk 설정 내)
  ☐ 월간 수익 목표: +10% 이상
```

## ⚠️ 위기 대응 프로토콜

### 문제: "거래 신호 없음"
```
원인 분석:
  1. 외부 신호가 약함 (current: -0.03)
  2. 시장이 횡보 중
  3. 리스크 필터 활성화

대응:
  ✓ 정상 (무리하지 말기)
  ✓ 대기하기 (24-48시간)
  ✓ 일일 로그로 신호 추적
```

### 문제: "연속 손실 (3연패 이상)"
```
원인 분석:
  1. 시장 레짐 변화
  2. 신호 신뢰도 낮음
  3. 변동성 과다

대응:
  ✓ 자동 쿨다운 (30분)
  ✓ 레버리지 50% 감소
  ✓ 신호 신뢰도 낮춤
```

### 문제: "API 연결 끊김"
```
자동 복구:
  1. 3회 재시도 (30초 간격)
  2. 기존 포지션 유지
  3. 로그 기록: [Exchange] 연결 실패

수동 복구:
  $ systemctl restart autotrader  # (Linux)
  $ brew services restart autotrader  # (macOS)
```
